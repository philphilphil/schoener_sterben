# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dark color scheme with OS-preference detection and a manual toggle in the header nav.

**Architecture:** Override CSS custom properties via `html[data-theme="dark"]` selector. An inline blocking script in `<head>` reads localStorage / `prefers-color-scheme` before first paint to prevent FOUC. A toggle button in the header switches themes and persists the choice.

**Tech Stack:** CSS custom properties, vanilla JS (inline `<script>`), Astro components

## Chunk 2: Theme Init Script & Toggle Component

### Task 3: Add inline theme-init script to BaseLayout

The script must run before first paint to prevent flash of wrong theme.

**Files:**
- Modify: `src/layouts/BaseLayout.astro:16-17`

- [ ] **Step 1: Add inline script in `<head>`**

Insert immediately after `<html lang="de">` and before `<head>`:

Actually — since the script sets `data-theme` on `<html>`, it must go inside `<head>` as early as possible. Insert as the very first child of `<head>`, before the charset meta:

```html
<script is:inline>
  (function() {
    var saved = localStorage.getItem('theme');
    var theme = saved || (matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
    document.documentElement.setAttribute('data-theme', theme);
  })();
</script>
```

Using `is:inline` prevents Astro from processing/bundling the script — it stays as a blocking inline script.

- [ ] **Step 2: Make theme-color meta tag dynamic**

Replace the hardcoded `<meta name="theme-color" content="#fafafa">` at line 33 with:

```html
<meta name="theme-color" content="#fafafa" id="meta-theme-color">
```

The toggle script will update this dynamically.

```astro
    <line x1="1" y1="12" x2="3" y2="12"/>
    <line x1="21" y1="12" x2="23" y2="12"/>
    <line x1="4.22" y1="19.78" x2="5.64" y2="18.36"/>
    <line x1="18.36" y1="5.64" x2="19.78" y2="4.22"/>
  </svg>
  <svg class="icon-moon" xmlns="http://www.w3.org/2000/svg" width="18" height="18" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
    <path d="M21 12.79A9 9 0 1 1 11.21 3 7 7 0 0 0 21 12.79z"/>
  </svg>
</button>

<script is:inline>
  (function() {
    var btn = document.querySelector('.theme-toggle');
    if (!btn) return;

    function update() {
      var theme = document.documentElement.getAttribute('data-theme');
      btn.querySelector('.icon-sun').style.display = theme === 'dark' ? 'block' : 'none';
      btn.querySelector('.icon-moon').style.display = theme === 'dark' ? 'none' : 'block';
      var meta = document.getElementById('meta-theme-color');
      if (meta) meta.setAttribute('content', theme === 'dark' ? '#141414' : '#fafafa');
    }

    btn.addEventListener('click', function() {
      var current = document.documentElement.getAttribute('data-theme');
      var next = current === 'dark' ? 'light' : 'dark';
      document.documentElement.setAttribute('data-theme', next);
      localStorage.setItem('theme', next);
      update();
    });

    update();
  })();
</script>
```

## Chunk 3: Polish & Edge Cases

### Task 6: Handle OS preference changes

**Files:**
- Modify: `src/components/ThemeToggle.astro` (script section)

- [ ] **Step 1: Add matchMedia listener**

In the ThemeToggle script, add a listener so that if the user hasn't explicitly chosen a theme (no localStorage value) and they change their OS preference, the site updates:

Add after the `update()` call at the end of the script:

```js
matchMedia('(prefers-color-scheme: dark)').addEventListener('change', function(e) {
  if (!localStorage.getItem('theme')) {
    var theme = e.matches ? 'dark' : 'light';
    document.documentElement.setAttribute('data-theme', theme);
    update();
  }
});
```
